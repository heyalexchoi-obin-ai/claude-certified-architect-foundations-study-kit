## The 12 heuristics, one-liners (memorize before the exam)

1. Correctness-critical rule → **code/hook/gate**, not prompt.
2. Mis-selected tools → **fix descriptions first** (primary selection lever).
3. Calibration → **explicit criteria + few-shot**, not confidence/sentiment/ML.
4. **Scope decides location**: shared→project (`.claude/`, `.mcp.json`); personal→user (`~/.claude/`).
5. **Plan mode** = complex/architectural/multi-file; **direct** = simple/scoped. Plan *now* if complexity is known.
6. Cross-directory conventions → **`.claude/rules/` glob path-scoping**.
7. Coverage gaps → **coordinator's decomposition**, not the (correctly-scoped) subagents.
8. Failures → **structured error context** (type, query, partial results, alternatives).
9. **Least privilege**: scoped tool for the common case, coordinator for the rare complex case. Too many tools hurts selection.
10. CI/headless → **`claude -p`** (`--print`); `--output-format json` + `--json-schema` for structured output.
11. **Batch API** (50% off, ≤24h, no SLA) = non-blocking only; **sync** for blocking/pre-merge.
12. Big reviews → **multi-pass** (per-file + cross-file) and **independent** review instance, not single-pass/bigger-model/consensus.
